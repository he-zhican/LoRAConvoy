# \# LoRAConvoy

# 

# > 考虑通信有损情况的基于 LoRA 微调的多车道车队编队控制

# 

# 本仓库对应毕业论文第四章“通信有损场景下基于 LLM 的编队鲁棒控制方法”，主要实现了在 V2V 通信丢包条件下，利用历史状态缓存、邻车状态推断模型（NVSIM）和编队决策模型（CDM）完成车队编队控制的闭环流程。

# 

# \## 项目简介

# 

# 在多车道高速车队场景中，通信丢包会导致邻车状态信息缺失，进而影响编队决策与轨迹规划的稳定性。本项目围绕这一问题，构建了一个面向通信有损场景的鲁棒控制框架：

# 

# \- 通过 \*\*感知与通信模块\*\* 采集自车、邻车和环境信息；

# \- 通过 \*\*历史状态缓存与邻车状态推断模块\*\* 补全丢失的邻车状态；

# \- 通过 \*\*编队高层决策与控制模块\*\* 生成高层动作并驱动车辆执行；

# \- 通过 \*\*模型微调模块\*\* 使用 LoRA 对基础大模型进行轻量化任务适配。

# 

# \## 与论文第四章的对应关系

# 

# \- \*\*4.2 方法框架设计\*\*：对应本仓库的整体推理与控制流程。

# \- \*\*4.3 数据集构建\*\*：对应 `dataset/` 中的两类定向数据。

# \- \*\*4.4 模型微调与训练策略\*\*：对应基于 LLaMA3-8b 的 LoRA 微调方案。

# \- \*\*4.5 仿真实验与结果分析\*\*：对应 SUMO 仿真运行、日志记录与精度评估输出。

# \## 核心功能

# 

# \### 1. 邻车状态推断（NVSIM）

# 

# 当 V2V 通信发生丢包时，系统会利用邻车历史状态序列推断当前缺失状态，包括：

# 

# \- 位置

# \- 速度

# \- 车道

# 

# 推断结果会与实时可观测状态融合，形成更完整的场景表征，供后续决策模块使用。

# 

# \### 2. 编队决策（CDM）

# 

# 在补全场景信息后，系统调用编队决策模型生成高层动作，动作集合包括：

# 

# \- 左变道

# \- 右变道

# \- 保持当前状态（IDLE）

# \- 加速

# \- 减速

# 

# \### 3. SUMO 闭环仿真

# 

# 仓库中的仿真流程会在 SUMO 环境中持续执行决策、环境步进、碰撞检测和结果记录，并输出：

# 

# \- 仿真视频

# \- 运行日志

# \- 单次实验精度报告

# \- 全部实验汇总报告

# 

# \## 数据集说明

# 

# `dataset/` 目录下存放的是面向两类任务构建的定向数据：

# 

# \### 邻车状态推断数据集

# 用于训练模型根据历史时序状态推断当前邻车状态。样本通常采用 `instruction-input-output` 三段式格式：

# 

# \- `instruction`：说明推断任务

# \- `input`：历史若干时间步的状态序列

# \- `output`：当前时刻真值状态 JSON

# 

# \### 编队决策数据集

# 用于训练模型在通信有损场景下生成高层动作决策。样本同样采用 `instruction-input-output` 格式：

# 

# \- `instruction`：说明安全优先、编队协同等任务约束

# \- `input`：结构化场景描述

# \- `output`：唯一动作标签或动作 ID

# 

# \## 模型与训练策略

# 

# 本项目基于 \*\*LLaMA3-8b\*\* 进行两次独立的 LoRA 微调：

# 

# 1\. \*\*NVSIM\*\*：邻车状态推断模型

# 2\. \*\*CDM\*\*：编队决策模型

# 

# \### 训练特点

# 

# \- 冻结基础模型主干参数

# \- 仅训练低秩适配器参数

# \- 单卡训练友好

# \- 适合通信有损场景下的任务化适配

# 

# \### 常用训练参数

# 

# 以下参数与论文第四章保持一致：

# 

# \- `cutoff\_len = 1024`

# \- `per\_device\_train\_batch\_size = 2`

# \- `gradient\_accumulation\_steps = 8`

# \- `learning\_rate = 5e-5`

# \- `num\_train\_epochs = 8`

# \- `lora\_r = 8`

# \- `eval\_steps = 50`

# \- `save\_steps = 100`

# \- `val\_size = 0.1`

# \- `fp16 = True`

# 

# \## 配置说明

# 

# `config.yaml` 中主要配置项如下：

# 

# \- `OPENAI\_KEY`：大模型服务密钥

# \- `DECISION\_MODEL`：编队决策模型名称

# \- `PREDICT\_MODEL`：邻车状态推断模型名称

# \- `OPENAI\_API\_BASE`：模型服务地址

# \- `SUMO\_HOME`：SUMO 安装路径

# \- `episodes\_num`：实验轮数

# \- `result\_folder`：结果保存目录

# \- `simulation\_duration`：单次仿真步数

# \- `all\_vehicles\_decision`：是否对所有车都执行决策

#

# \## 注意事项

#

# \- 仿真输出与视频生成会占用一定磁盘空间，建议提前预留结果目录。
